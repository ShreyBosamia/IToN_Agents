# ADR-001: Use Firecrawl-First Hybrid Web Scraping

**Status:** Accepted (Updated 2026-04-22)

## Context

Our project requires extracting data from many third-party websites with inconsistent structures and frequent use of JavaScript for rendering content. We need a scraping method that reliably handles dynamic pages, reduces local browser-management overhead, and still preserves a strong fallback path for sites where managed scraping underperforms.

## Decision

We will use **Firecrawl as the default managed scrape provider** for general webpage retrieval, with **Playwright (headless Chromium) as an automatic fallback** and as the preferred option for interactive or click-driven directory flows.

The public `scrape_website` tool contract remains stable. Provider routing is controlled by `SCRAPER_PROVIDER`:

- `auto` (default): Firecrawl first, Playwright fallback
- `firecrawl`: Firecrawl only
- `playwright`: Playwright only

## Options Considered

- **Do nothing (basic HTTP fetch)**
  - Simple but fails on JavaScript-rendered content.

- **Firecrawl-first hybrid (chosen)**
  - Lower operational burden for common scraping, managed handling of many dynamic/blocked pages, and a clean fallback path.

- **Playwright only**
  - Reliable, but higher local runtime/CI burden and more direct browser-management overhead.

- **Puppeteer**
  - Similar, but Playwright offers better cross-browser support and tooling.

- **Selenium/WebDriver**
  - Heavier and slower for our use case.

- **Scraping APIs (Zyte, Diffbot, etc.)**
  - Lower maintenance but costly and inflexible.

## Consequences

### Positive

- Reduces dependence on locally managed browser execution for common scrape cases.
- Preserves Playwright reliability for dynamic or difficult sites.
- Keeps existing downstream extraction and pipeline consumers unchanged.
- Makes provider choice explicit and auditable in scrape output.

### Negative

- Introduces vendor cost and another external dependency.
- Requires fallback heuristics and provider-specific normalization.
- Still requires Playwright to remain installed for fallback and directory flows.
